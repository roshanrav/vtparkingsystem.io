1. Real-Time Data Module

The Real-Time Data Module is responsible for continuously collecting, processing, and displaying real-time parking availability information. It gathers data from parking sensors that are distributed in different parking lots. This module employs Process Control Architecture, handling data flow from sensors and updating the availability status in real time for display to users. Allowing availability information to be displayed dynamically for users of the system enables them to effectively locate available parking spots in a fraction of the time needed previously.

2. Reservation and Payment Module

The Reservation and Payment Module allows drivers to reserve parking spots and pay electronically. The module contains secure infrastructure through a gateway to maintain a safe environment for financial transactions. In addition, notifications are configured to work within the system to inform users of current reservation status, confirmation of payments, and any parking or payment cancellations in real time.

3. Analytics and Administration Module

The Analytics and Administration Module allows parking administrators to overview current parking status and overall data analysis of the parking system. This module aggregates data on parking patterns, peak times, and ticket frequency to better allow for resource and personnel allocation to handle day to day parking needs of drivers on campus. The interface is user-friendly and easy to navigate for ease of use by administration, with the ability to adjust visible metrics depending on what is needed at any moment.
