Hokie Parking System Architecture

Designing a top-level architecture for the Hokie Parking System requires a balance between managing real-time data, user interactions (including users like drivers and administrative team members), and security concerns. To address the full scope of the problem, we decided to break down the Hokie Parking System into three core functional modules. This modular approach allows us to select an appropriate architectural style for each module and enables us to design a system that meets both functional and non-functional requirements.
